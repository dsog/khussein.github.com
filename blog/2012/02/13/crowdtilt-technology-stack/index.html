Crowdtilt Technology Stack

Feb 13th, 2012 | Comments

Crowdtilt is built using Perl and a plathora of CPAN modules such as ubic and Rex for service management and POE::Component::MessageQueue for queing. The database is PostgreSQL. We use Amazon SES for sending emails. All configurations are managed by Chef and all code is hosted at Github.

Production servers are hosted on EC2 with Ubuntu images. The webserver is Starman with nginx as a proxy and loadbalancer. For caching, we use memcached.

We maintain tasks, bugs, and feature requests in Trello. Team communication happens over emails, IRC, and Mumble. Development environments are built with Vagrant and identical to production setup. We use Jenkins for continuous integration.

Crowdtilt’s email is served by Google Apps. We also use Google Analytics for monitoring the traffic. The official blog is hosted on Tumblr.
